E1 Core MCP Client

Implementação do MCP Client com lifecycle completo: inicialização, capability negotiation e encerramento de conexão.

US01

Conexão STDIO com server local

Alta MVP

Como desenvolvedor, quero conectar o orcha-mcp a um MCP server local via STDIO, para ver o ciclo de inicialização JSON-RPC acontecendo.

  • Inicia o processo do server como subprocesso
  • Envia initialize com protocolVersion e clientInfo
  • Recebe resposta com capabilities do server
  • Envia notifications/initialized
  • Conexão encerrada graciosamente ao finalizar
US02

Conexão HTTP com server remoto

Alta MVP

Como desenvolvedor, quero conectar o orcha-mcp a um MCP server remoto via HTTP, para comparar o comportamento entre transports.

  • Faz POST para a URL do server com mensagem initialize
  • Suporta headers customizados (Bearer token)
  • Lida com respostas síncronas (200 JSON) e SSE
  • Timeout configurável (padrão: 10s)
US03

Exibição da capability negotiation

Alta MVP

Como desenvolvedor, quero ver no terminal as capabilities negociadas durante o handshake.

  • Exibe resumo formatado: nome, versão, capabilities
  • Com --verbose, exibe JSON-RPC raw do initialize
✓ filesystem v0.6.2 tools(listChanged=true)
✓ fetch v0.1.0 tools

E2 Configuração

Leitura, validação e gerenciamento do arquivo orcha.json.

US04

Configuração de múltiplos servers via JSON

Alta MVP
  • orcha.json lido da raiz por padrão
  • Suporte a type: "local" e type: "remote"
  • Campos: command, url, enabled, environment
  • Erros de schema com mensagem clara
US05

Ativar e desativar servers individualmente

Alta MVP
  • Server com "enabled": false não é inicializado
  • Flag --server <nome> restringe a operação
US06

Path alternativo de config

Média MVP
  • Flag --config <path> aceita absoluto ou relativo
  • Erro claro se arquivo não existir

US08

Inspecionar o JSON-RPC do tools/list

Alta MVP
  • Com --verbose, imprime mensagem e resposta pretty-printed
  • Prefixos de direção: → HOST→SERVER / ← SERVER→HOST
US09

Inspecionar o inputSchema de uma tool

Média MVP
  • Comando orcha describe <tool_name> exibe inputSchema completo
  • Indica campos required e opcionais

E4 Tool Execution

Execução direta de tools via CLI, sem passar pelo LLM.

US10

Executar uma tool diretamente

Alta MVP
  • Aceita nome com namespace (filesystem_read_file)
  • Argumentos como JSON string via --args
  • Retorna código 1 se a tool retornar erro
US11

Roteamento automático para o server correto

Alta MVP
  • Roteamento pelo prefixo do namespace (filesystem_*)
  • Erro claro se tool não for encontrada
  • Erro claro se houver ambiguidade
US12

Inspecionar JSON-RPC do tools/call

Alta MVP
  • Com --verbose, imprime request e response do tools/call
US13

Modo dry-run

Média MVP
  • Exibe JSON-RPC que seria enviado sem executar
  • Não faz conexão real com servers
